surfaceid.surfaceid.util.utils.search(o1, rho, list_indices, within, library, library_within, pdbs_unique, patch_sizes, pdbs, x_desc, idxs_desc, mini_bs, device, nmin_pts, expand_radius, nmin_pts_library, thres, target, idxs_contact)[source]
performs the search for all pairs between the query and candidate

first, it computes the descriptor distance between all pair of verticies between query and candidate. it selects pairs within the descriptor distance thresheold. if number of hits between query and candidate exceeds the threshold arameter, hit grouping is performed to unify vertex hits with their nearest neighbors and form a hit area on each protein surface.

Parameters

  • o1 (torch.tensor) – Surface ID descriptors for the query protein

  • rho (np.array) – radial distribution of vertices on a patch from the patch center

  • list_indices (np.array) – embedded list containing the vertex ids within each patch

  • within (np.array) – vertex ids that fall within the neighbor_dist for the query patch

  • library (np.array) – protein ids in the library

  • library_within (np.array) – embedded list of vertex ids corresponding to the vertices within the neighbor_dist of each patch in the candidate protein

  • pdbs_unique (np.array) – unique pdb ids

  • patch_sizes (np.array) – number of patches for every candidate protein within the area of interest

  • pdbs (np.array) – protein names corresponding to each patch in the library following the Surface ID descriptor

  • x_desc (torch.tensor) – Surface ID descriptors

  • idxs_desc (torch.tensor) – patch ids for the corresponding descriptors

  • mini_bs (torch.tensor) – mini batches to perform the search

  • device (str) – CPU or GPU

  • nmin_pts (int) – minimum number of hit patches on query for identifying it as a hit

  • expand_radius (float) – the expansion radius around the hit vertices to define the hit area following the clustering step

  • nmin_pts_library (int) – minimum number of hit patches on candidate protein for identifying it as a hit

  • thres (float) – threshld used for descriptor distance to identify two patches as similar

  • target (str) – protein id for the query

  • idxs_contact (np.array) – patch indices for located at the area of contact

Returns
